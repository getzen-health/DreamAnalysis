Device Adapter Architecture

// Every device implements WearableAdapter
interface WearableAdapter {
  name: string              // "whoop" | "oura" | "garmin" | ...
  authenticate(): Promise<OAuthTokens>
  sync(userId: string, since: Date): Promise<HealthSample[]>
  getCapabilities(): MetricType[] // what this device can provide
  webhookHandler?(req): void     // for push-based devices
}
 
// All adapters normalize to the same HealthSample format
interface HealthSample {
  source: "whoop" | "oura" | "garmin" | "apple" | "google" | "cgm" | "eeg" | "manual"
  metric: "heart_rate" | "hrv_rmssd" | "resting_hr" | "respiratory_rate" | "spo2" |
          "skin_temp" | "sleep_deep_min" | "sleep_rem_min" | "sleep_light_min" |
          "sleep_efficiency" | "steps" | "active_calories" | "weight_kg" |
          "body_fat_pct" | "vo2_max" | "workout_strain" | "glucose_mgdl" | ...
  value: number
  unit: string
  recorded_at: Date
  metadata?: Record<string, unknown>
}
 
// Pipeline doesn't care where data comes from
// Same health_samples table, same triggers, same scores
ingestHealthData(samples: HealthSample[]) → pipeline handles the rest

OAuth Flow (Whoop, Oura, Garmin, CGM)

1. User clicks "Connect Whoop" in Settings
2. Redirect → api.prod.whoop.com/oauth/oauth2/auth
3. User authorizes → callback with auth code
4. Edge Function: exchange code → access + refresh tokens
5. Store tokens in device_connections table (encrypted)
6. Trigger initial historical sync (last 30 days)
7. Set up recurring sync (15-min polling) + webhook registration
8. All data → normalized HealthSample[] → pipeline

device_connections (new table)

id uuid PK
user_id uuid FK → users
provider text (whoop | oura | garmin | dexcom | libre | strava)
access_token text (encrypted)
refresh_token text (encrypted)
token_expires_at timestamptz
scopes text[]
last_sync_at timestamptz
sync_status text (active | error | disconnected)
error_message text
connected_at timestamptz
UNIQUE(user_id, provider)

Data Source Priority (When Multiple Devices Report Same Metric)

If user wears both Whoop AND Oura, who wins for HRV?

1. User preference — user picks primary device per metric in Settings
2. Highest fidelity — if no preference, use device with most data points
3. Recency — if tied, use most recent reading
4. All stored — every source writes to health_samples regardless, priority only affects score computation
