How do we get to Supabase?

One critical decision before I design the full system.

A

Big Bang Migration

Move everything to Supabase at once. Replace Neon, Express auth, file storage in one go. Then build new features on top.

Pros

  • Clean cut — no dual-database complexity
  • All new features built on final architecture

Cons

  • High risk — everything breaks at once
  • Weeks before new features start
  • Existing features may regress
B

New Features on Supabase, Old Stays on Neon

Keep existing Neon DB for current features (EEG, dreams, food, auth). Build ALL new Bevel features (pipeline, scores, exercise, weight, habits) directly on Supabase. Migrate old features later when stable.

Pros

  • Zero risk to existing features
  • Start building new stuff immediately
  • Migrate old features at your pace
  • Can validate Supabase before committing fully

Cons

  • Two databases temporarily
  • Cross-DB queries need API bridge
C

Fresh Start on Supabase

Rebuild the entire app from scratch on Supabase. New schema, new API, new frontend. Keep ML backend as-is.

Pros

  • Cleanest architecture
  • No legacy baggage

Cons

  • Massive effort — weeks to rebuild existing features
  • Loses all existing work on 46 pages, 3200-line routes.ts
  • High risk of scope creep

My Recommendation: Option B

Build all new Bevel features on Supabase from day one. Existing EEG/dream/food features keep working on Neon untouched. You get to start coding the pipeline immediately without breaking anything. Here's what goes where:

Stays on Neon (existing)
  • Users + Auth (bcrypt sessions)
  • EEG sessions + emotion readings
  • Dream analysis + symbols
  • Food logs + meal history
  • AI chat history
  • Study protocol tables
New on Supabase (Bevel features)
  • Health data pipeline + ingestion
  • Score engines (Recovery, Sleep, Strain, Stress, Energy Bank)
  • Weight + body composition tracking
  • Exercise + workout logging
  • Habit journal
  • Cycle tracking
  • Trend detection + daily aggregates
  • Smart alarm logic
  • Nutrition score (AHEI)
  • Cardio load / HR zones

Later (when ready): migrate auth → Supabase Auth, move EEG/dream tables, retire Neon. No rush.
