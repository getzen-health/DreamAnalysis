Recommended Stack

Frontend

Vercel (Free)

React dashboard. Already deployed. $0.

Database

Supabase (Free Tier)

PostgreSQL + real-time + auth + storage. 500MB DB, 1GB files. Replaces Neon.

Health API

Supabase Edge Functions (Free)

Serverless API for health data ingestion, score computation, trend detection. 500K invocations/mo free.

ML Backend

Local FastAPI (Free)

16 ML models + EEG stream + BrainFlow. Stays on Mac. Writes to Supabase DB.

Event Bus

Supabase Realtime (Free)

Postgres LISTEN/NOTIFY replaces Redis Streams. Dashboard auto-updates when scores change. No Redis needed.

Score Engine

PostgreSQL Triggers + Cron

DB triggers fire score recomputation on every INSERT. pg_cron for daily aggregates. Scores live IN the database.

Why This Is "Very Very Strong"

Event-Driven

Every health data INSERT triggers score recomputation via Postgres triggers. No polling.

Real-Time Delivery

Supabase Realtime pushes score changes to all connected clients instantly. No refresh needed.

Always-On Health Tracking

Cloud API receives health data 24/7, even when Mac is off. EEG adds depth when you're at your desk.

$0 Monthly Cost

Supabase free tier + Vercel free tier + local ML. No forgotten bills. No Render surprises.

Trend Detection in DB

PostgreSQL window functions for moving averages, rate-of-change, anomaly detection. Pre-computed daily rollups via pg_cron.

Scales When Ready

If you outgrow free tier, Supabase Pro is $25/mo for 8GB. Add Redis later only if needed.

What Changes From Current Setup

Database Neon PostgreSQL → Supabase PostgreSQL Same Drizzle ORM, just change connection string
Auth bcrypt + express-session → Supabase Auth Moderate migration, but gains OAuth/magic link free
Real-time Custom WebSocket → Supabase Realtime (health only) EEG WebSocket stays as-is
File Storage None / local → Supabase Storage Dream images, food photos, voice recordings
Express Server Cloud (Railway) → Local + Edge Functions Complex routes stay local, simple CRUD moves to Edge

Alternative: Stay on Neon (Lower Risk)

If Supabase migration feels too risky, we can keep Neon PostgreSQL and build the pipeline with:

  • PostgreSQL triggers for score computation (works on Neon too)
  • Server-Sent Events (SSE) instead of Supabase Realtime
  • Keep Express on Railway or run locally
  • Skip auth migration entirely

Trade-off: No free real-time subscriptions, no built-in storage, no auth upgrade. Same pipeline logic, just more custom code.
