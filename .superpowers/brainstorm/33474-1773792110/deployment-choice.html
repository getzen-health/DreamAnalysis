Where does this run?

This determines the entire pipeline backbone — database, event bus, real-time layer, and cost.

A

Local Development Only

Runs on your Mac. No cloud costs. Redis + TimescaleDB locally. Best for prototyping, but no mobile access.

Pros

  • $0 cost
  • Full control
  • Any database/tool

Cons

  • No mobile sync
  • Not shareable
  • Must be running
B

Vercel + Railway

Your current pattern. Vercel frontend, Railway for Express + FastAPI backends, Neon PostgreSQL.

Pros

  • Familiar stack
  • Railway supports Redis
  • Real deployment

Cons

  • Railway costs ~$5-20/mo
  • Neon free tier: 0.5GB storage
  • No TimescaleDB on Neon
C

Vercel + Supabase

Supabase gives you PostgreSQL + real-time subscriptions + auth + storage on free tier. No separate Redis needed.

Pros

  • Free tier: 500MB DB + 1GB storage
  • Built-in real-time (Postgres LISTEN/NOTIFY)
  • Row-level security
  • Edge Functions for serverless

Cons

  • New integration work
  • No TimescaleDB extension
  • WebSocket streaming needs separate host
D

Hybrid: Local ML + Cloud Data

FastAPI ML backend runs locally (EEG needs hardware). Express + DB in cloud. Best of both worlds.

Pros

  • ML models stay on device (fast)
  • Data accessible from anywhere
  • Mobile health sync works remotely
  • EEG hardware needs local anyway

Cons

  • Two environments to manage
  • Cloud DB costs if heavy usage
