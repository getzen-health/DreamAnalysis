1. Brain Wave Bands

Your brain produces electrical oscillations at different frequencies. Different mental states produce different dominant frequencies. EEG measures these oscillations via electrodes on your scalp.

Delta (δ)
0.5 – 4 Hz

Deep unconscious sleep (N3 stage). Highest amplitude waves. Virtually absent during wakefulness. High delta = deep sleep / unconscious processing.

Deep sleepN3 stage
Theta (θ)
4 – 8 Hz

Drowsiness, light sleep, creativity, deep meditation. Frontal midline theta increases with memory encoding and working memory load. REM sleep shows theta bursts.

CreativityMemoryMeditation
Alpha (α)
8 – 12 Hz

The "idle" rhythm. Strongest when eyes are closed, mind is relaxed, not actively processing. Suppressed (desynchronized) when you engage in a task. Alpha asymmetry between left and right hemispheres encodes emotional valence — this is the FAA signal.

RelaxationFAA valenceIdle state
Beta (β)
12 – 30 Hz

Active thinking, concentration, anxiety, stress. Low-beta (12–20 Hz) = task focus. High-beta (20–30 Hz) = anxiety, worry, fight-or-flight. Beta increases when alpha suppresses (inverse relationship).

FocusAnxietyStress
Gamma (γ)
30 – 100 Hz

⚠ Mostly EMG artifact on Muse 2! In research settings with gel electrodes: high-level cognition, binding of sensory information, lucid dreaming, advanced meditation. On Muse 2's dry electrodes at AF7/AF8, gamma is primarily frontalis muscle noise from jaw clenching, forehead tension.

⚠ EMG artifactMeditation (rare)
The Gamma Problem on Muse 2: The frontalis muscle sits directly beneath the AF7 and AF8 electrodes. Any jaw clenching, eyebrow raising, or forehead tensing generates broadband muscle noise at 20–100 Hz. This is physically indistinguishable from neural gamma using consumer dry electrodes. This is why gamma has been removed from arousal, stress, and focus formulas.

3. Signal Processing Pipeline

Raw EEG from Muse 2 goes through this chain before any ML model sees it:

1
Raw Signal In — (4, n_samples) float32 array at 256 Hz, values in µV.
Typically noisy, contains 50/60 Hz powerline interference, drift, artifacts.
2
Butterworth Bandpass Filter — 1–50 Hz, order=5, filtfilt (zero-phase)
Removes DC drift (below 1 Hz) and high-frequency noise above 50 Hz. Zero-phase means no temporal distortion.
3
Notch Filters — 50 Hz + 60 Hz
Removes powerline interference (50 Hz in Europe/Asia, 60 Hz in Americas).
4
Welch PSD — Power Spectral Density via Welch's method.
Computes how much power exists at each frequency. Far more stable than raw FFT. Computed once per epoch and shared to all sub-extractors.
5
Band Power Extraction — Integrates PSD under each band (delta/theta/alpha/beta/gamma).
Also computes log-domain powers (better statistical normality for ML).
6
Feature Vector (17 features): band powers, log powers, ratios (α/β, θ/β, α/θ), Hjorth parameters (activity, mobility, complexity), spectral entropy, differential entropy per band.
7
FAA Computation — Frontal Alpha Asymmetry across AF7 and AF8.
Called separately and blended into valence signal.
8
ML Inference — Feature vector fed to the appropriate model (heuristic, ONNX, sklearn, multichannel).

Code Location

  preprocess(signal, fs)           → bandpass + notch filters
  extract_band_powers(signal, fs)  → delta/theta/alpha/beta/gamma

Hjorth Parameters (Bonus Features)

Hjorth Activity — signal variance (power estimate) Activity = Var(x) Hjorth Mobility — normalized standard deviation of first derivative Mobility = sqrt(Var(x') / Var(x)) Hjorth Complexity — ratio of signal's complexity to pure sine wave Complexity = Mobility(x') / Mobility(x) High complexity = irregular signal (artifact, muscle). Low complexity = rhythmic (alpha, sleep spindles).

4. Frontal Alpha Asymmetry (FAA)

FAA is the most scientifically validated EEG marker for emotional valence. Established by Richard Davidson at UW-Madison (1992) and replicated hundreds of times.

The Core Principle

Left frontal regions are associated with approach motivation (positive affect, wanting, engagement). Right frontal regions are associated with withdrawal motivation (negative affect, avoidance, depression).

Alpha is inversely related to activation — more alpha = less activity. So:

  • More left-frontal alpha (AF7 ↑) = left hemisphere less active = more withdrawal = negative valence
  • More right-frontal alpha (AF8 ↑) = right hemisphere less active = less withdrawal = positive valence

Therefore: FAA = log(right_alpha) − log(left_alpha). Positive = positive affect.

Frontal Alpha Asymmetry (Davidson, 1992) FAA = ln(AF8_alpha) − ln(AF7_alpha)
Map to valence signal (−1 to +1) asymmetry_valence = clip( tanh(FAA × 2.0), −1, +1 )
FAA > 0 → positive valence (happy, calm, approach state) FAA < 0 → negative valence (sad, anxious, withdrawal state) tanh(x × 2.0) gives smooth S-curve: small FAA → small valence shift; large FAA → near ±1

Concrete Example

Scenario: You're feeling anxious and worried about a deadline.

  • Right frontal cortex activates (withdrawal/avoidance motivation) → AF8 alpha suppresses
  • AF8_alpha = 0.3 µV², AF7_alpha = 0.6 µV²
  • FAA = ln(0.3) − ln(0.6) = −1.2 − (−0.5) = −0.7
  • asymmetry_valence = clip(tanh(−0.7 × 2.0), −1, 1) = tanh(−1.4) ≈ −0.885
  • Result: Strong negative valence → system reports sad/fearful state ✓

Scenario: You just solved a hard problem and feel satisfied.

  • Left frontal cortex activates → AF7 alpha suppresses
  • AF7_alpha = 0.2 µV², AF8_alpha = 0.5 µV²
  • FAA = ln(0.5) − ln(0.2) = −0.69 − (−1.61) = +0.92
  • asymmetry_valence = clip(tanh(0.92 × 2.0), −1, 1) = tanh(1.84) ≈ +0.951
  • Result: Strong positive valence → system reports happy state ✓
Critical Bug That Was Fixed: The old predict() function was passing only eeg[0] (AF7 alone) to _predict_features(), discarding AF8/TP9/TP10. FAA was computed as log(AF7) − log(AF7) = 0 always. Every single emotion reading had zero FAA contribution. Fixed in commit 270d27e.

5. Russell's Circumplex Model of Emotion

All emotions live in a 2D space defined by Valence (positive/negative feeling) and Arousal (energetic/calm). This model, proposed by James Russell (1980), is the most widely used framework for EEG-based emotion recognition.

HIGH AROUSAL (energetic) Angry Excited Enthusiastic ↖ ↑ ↗ NEGATIVE ──────────────────────────────── POSITIVE ↙ ↓ ↘ Sad/Dep Tired Content/Calm LOW AROUSAL (calm)

Valence (X-axis)

How positive or negative the feeling is. −1 = most negative (deep sadness/fear), +1 = most positive (joy/love).

Best EEG predictor: FAA (frontal alpha asymmetry)

Accuracy: ~60–70% (difficult)

Arousal (Y-axis)

How energized or calm. 0 = completely calm/asleep, 1 = maximally alert/activated.

Best EEG predictor: β/(α+β) ratio

Accuracy: ~70–85% (easier)

6. All Formulas Used in the System

These are the exact formulas in ml/models/emotion_classifier.py after the Feb 2026 fixes.

Band Power Normalization

Relative band powers (each band as fraction of total) total = delta_raw + theta_raw + alpha_raw + beta_raw + gamma_raw alpha = alpha_raw / total (and similarly for each band) This normalization makes values comparable across people and sessions.

Valence Formula

Step 1: Alpha/Beta Ratio valence component valence_abr = tanh( (α/β − 0.7) × 2.0 ) α/β ratio > 0.7 → positive; < 0.7 → negative. tanh gives smooth −1 to +1 output. Step 2: FAA component (when 2+ channels available) faa_valence = tanh( FAA × 2.0 ) where FAA = ln(AF8_alpha) − ln(AF7_alpha) Step 3: Blend (50/50 when multichannel) valence = clip(0.50 × valence_abr + 0.50 × faa_valence, −1, +1) Single-channel fallback: valence = clip(0.65×tanh((α/β−0.7)×2) + 0.35×tanh((α−0.15)×4), −1, +1)

Arousal Formula

Three-component arousal (gamma REMOVED — was EMG noise) arousal = clip( 0.45 × β/(β+α) ← beta dominance + 0.30 × (1 − α/(α+β+θ)) ← inverse alpha fraction + 0.25 × (1 − δ/(δ+β)), ← inverse delta (awake proxy) 0, 1) Delta inclusion: deep relaxation/sleep has high delta; adding inverse delta suppresses arousal when sleeping.

Stress Index

Stress from high-beta, beta asymmetry, theta/alpha ratio stress = clip( 0.45 × high_beta/(β+α) ← high-beta (anxiety band) dominance + 0.30 × (θ/α) × 0.3 ← theta/alpha ratio (stress marker) + 0.25 × high_beta_frac, ← fraction of beta that is high-beta 0, 1)

Focus Index

Focus from beta/alpha ratio and alpha suppression focus = clip( 0.45 × β/(α+β) ← beta dominance + 0.40 × (1 − α/(α+β)) ← alpha suppression + 0.15 × (1 − θ_β_ratio × 0.2), ← low theta (not drowsy) 0, 1)

Emotion Probabilities

Happy probability (valence+, moderate arousal, alpha present) p_happy = 0.45×max(0, valence) + 0.30×max(0, arousal−0.3) + 0.25×min(1, α/β×0.8) Sad probability (negative valence, low arousal, high theta) — threshold −0.10 not −0.25 p_sad = 0.50×max(0, −valence−0.10) + 0.30×max(0, 0.35−arousal) + 0.20×max(0, θ/β−2.0)×0.4 Angry probability (negative valence, high arousal, high beta) p_angry = 0.40×max(0, −valence−0.1) + 0.30×max(0, arousal−0.45) + 0.25×min(1, β/α−1.3) + 0.05×min(1, γ×2.5) Fear (more extreme than angry — very high arousal + very negative valence) p_fear = 0.45×max(0, −valence−0.15) + 0.35×max(0, arousal−0.5) + 0.20×min(1, β/α−1.5) Surprise (high arousal regardless of valence) p_surprise = 0.70×max(0, arousal−0.6) + 0.30×max(0, 1−α_power×3)

EMA Smoothing (Anti-flicker)

Exponential Moving Average — applied after each prediction output_t = α_ema × raw_t + (1 − α_ema) × output_(t-1) α_ema = 0.35 (responds in ~3 frames to a step change) Without EMA: emotion label flickers every frame. With EMA: smooth, stable readings.

Softmax Temperature Scaling

Applied to probability vector to sharpen predictions p_i = exp(logit_i / T) / sum(exp(logit_j / T)) T = 2.5 (temperature parameter) T > 1 = softer distribution (less confident). T < 1 = sharper (more winner-takes-all). T=2.5 smooths out overconfident raw scores.

Why Cross-Subject EEG is Hard

  • Within-subject accuracy: ~90.97% (same person, same day)
  • Cross-subject accuracy: ~64.82% (different person) — 26 point drop
  • Each person's brain has different anatomy → slightly different electrode positions over different cortical regions
  • Individual differences in alpha frequency (8–12 Hz, varies ±1.5 Hz per person)
  • Solution: baseline normalization (+15–29% accuracy in literature)

Impact: CRITICAL. This caused FAA to always be zero.

Fix 2: Gamma EMG Contamination Removed

Impact: HIGH. Jaw clenching was falsely reported as high arousal, stress, anger, focus.

# BEFORE — gamma included as neural signal
arousal_raw = 0.35*β/(β+α) + 0.30*(1-α/(α+β+θ)) + 0.35*γ
stress_index = 0.40*high_β + 0.35*θ/α + 0.25*γ
# AFTER — gamma removed, delta added as sleep/relaxation inverse
arousal_raw = 0.45*β/(β+α) + 0.30*(1-α/(α+β+θ)) + 0.25*(1-δ/(δ+β))
stress_index = 0.45*high_β/(β+α) + 0.30*(θ/α)*0.3 + 0.25*high_β_frac

Fix 3: Sad Never Triggered (Threshold Too High)

Impact: MEDIUM. At neutral resting state (valence≈0), sad was essentially impossible.

# BEFORE — required valence < -0.25 before any sad probability
p_sad = 0.50 * max(0, -valence - 0.25) + ...
# AFTER — triggers as soon as valence goes slightly negative
p_sad = 0.50 * max(0, -valence - 0.10) + ...

Fix 4: Duplicate Arousal Computation in _predict_onnx

Impact: MEDIUM. Lines 509 and 512 both computed arousal identically — second silently overwrote first.

Fixed: removed both instances, replaced with single corrected computation using new formula.

Fix 5: Single-Term Valence in ONNX/Sklearn Paths

# BEFORE — one term
valence = tanh((alpha/beta - 0.7) * 2.0)
# AFTER — two terms: ratio + absolute alpha level
valence = 0.65*tanh((alpha/beta - 0.7)*2.0) + 0.35*tanh((alpha - 0.15)*4)

The second term catches cases where α/β ratio is high but absolute alpha is low (stressed state).

10. Future Work (Priority Order)

Priority 1: Baseline Calibration Protocol

Expected improvement: +15–29% accuracy (published in multiple papers)

At session start, record 2 minutes of resting-state EEG (eyes closed, relaxed). Compute personal baseline band powers. Normalize all subsequent readings against this baseline.

baseline_alpha = mean(alpha_power_over_2min)
normalized_alpha = current_alpha / baseline_alpha
# Now comparable across people and sessions

This eliminates individual differences in absolute power levels — the biggest source of cross-subject error.

Priority 2: Longer Epoch Windows

Current: 1-second windows. Problem: Welch PSD at 256 Hz needs >4 seconds for stable spectral estimates at 1 Hz frequency resolution.

Fix: Use 4-second sliding window with 50% overlap (2-second step). This gives stable PSD with 0.25 Hz frequency resolution.

# Welch PSD frequency resolution = fs / n_samples
# 1 sec: 256 samples → 1 Hz resolution (too coarse to distinguish alpha from theta)
# 4 sec: 1024 samples → 0.25 Hz resolution (accurately separates 8-12 Hz alpha)

Priority 3: Personal Calibration via Online Learner

After 5+ sessions, compute per-user alpha frequency peak (individual alpha frequency, IAF). Adjust band boundaries relative to IAF. The Online Learner (ml/models/online_learner.py) is ready — needs integration into live inference path in ml/api/routes.py.

Priority 4: Collect Muse 2-Specific Training Data

The DEAP model trained on 32-ch gel electrodes degrades massively on 4-ch dry electrodes (domain gap). Collecting 30+ minutes of labeled EEG from Muse 2 with ground-truth emotion labels (video-induced, self-report) would allow retraining a model native to the hardware.

Use /api/collect-training-data endpoint which is already implemented.

Quick Wins That Are Already Coded: The EMA smoothing, auto-calibration in StressDetector, and confidence/reliability endpoints in the API already exist. The main gaps are baseline normalization and epoch length — both achievable without new ML training.
